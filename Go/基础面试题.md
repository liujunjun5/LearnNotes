## defer是什么，有什么作用
defer 作为go中的一个关键字，后续的函数作为延迟完成的部份一定会执行。类似于java的finally。  
常用于资源关闭、锁释放、收尾工作。  
g中的一个指针字段标识了defer链表，用了头插法的方式，所以这里是栈类型，后进先出。
## defer执行顺序
return赋值（观察有无临时变量，有无闭包情况，有无参数传递），defer执行，返回。  

## Go对比其他语言（Java）
- 并发模型
  - g+管道 2kb消耗，轻松十万百万+
  - java 线程+锁，1MB消耗，上百到上千
- 垃圾回收
  - 三色标记+写屏障 吞吐量小，延迟小
  - java支持多种回收，自由选择
- 内存占用、启动
  - go小启动快 几MB
  - java内存占用JVM元空间堆 上百MB
- 生态
  - java spring规范，团队协作
  - go新兴的

## 协程、线程、进程
- 进程：一个拥有特定功能的程序，虚拟内存、文件描述符、通过进程间通信，稳定但是切换上下文消耗大，资源分配的最小单位
- 线程：CPU调度的最小单位，轻量级进程，多个线程可能在同一个进程中，主要通过共享内存的方式通信，有自己的栈空间、程序计数器，上下文切换较快
- 协程：轻量级线程，运行在用户态，不需要复杂的状态变化，内存占用小，可以轻松的创建十万百万
## new和make
- new 是分配内存，不初始化，可以运用在任何的类型中，地址
- make 是分配内存并且初始化，用于slice、map、channel中，值

## slice和数组的区别
数组是长度确定的，一般需要初始化时指定或者创建时有传入的数组计算出值
slice是对数组的封装，结构是len+array指针。
slice拥有自动扩容机制，又称不定数组。
